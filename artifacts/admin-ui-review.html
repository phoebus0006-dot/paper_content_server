修复的布局问题

  • 侧边栏导航 — 替换原来的无导航页面为左侧 220px 固定侧边栏，包含 6 个导航入口（总览/新闻审查/图片库/图片编辑/发布中心/运行状态）
  • 页面头部 — 每个页面顶部有统一的页头（标题 + 刷新时间 + 刷新按钮）
  • 统一卡片设计 — 所有区块使用统一的 card 组件（圆角 10px、边框、阴影、内边距）
  • 统计网格 — 使用 stat-grid-4 和 stat-grid-6 网格布局替代原来的随意排列
  • 新闻两栏布局 — 新闻审查页改为左栏列表 + 右栏详情的两栏布局
  • 图片网格 — 图片库改为自动填充网格卡片，每张图统一缩略图+信息+操作按钮
  • 发布历史表格 — 发布中心使用 grid 布局的行式卡片
  • 运行状态分区 — 运行状态页分为 5 个卡片（健康检查/Frame状态/内容缓存/请求统计/服务配置）
  • z-index 修复 — 主内容区添加 z-index:1 防止侧边栏遮挡按钮点击

修复的空态/错误态问题

  • 消除所有 -- 占位符 — 替换为有意义的中文空态文本（"暂无数据"/"未设置"/"未生成"/"未调度"等）
  • setText() 函数 — 新增统一文本设置函数，自动处理 undefined/null/空字符串/-/-- 为有意义的空态
  • 新闻空态 — 新闻列表为空时显示"暂无新闻数据"提示，而非空白
  • 图片空态 — 图片库为空时显示"图片库为空"提示
  • 加载失败空态 — 所有 API 加载失败时显示具体的错误信息，而非静默空白
  • 页面错误框 — 新增 page-error-box 组件，初始化异常时显示可见错误框，不白屏
  • Broken image 处理 — 图片加载失败时显示图标占位符，不显示 broken image
  • 健康检查 URL 修复 — 修正 loadHealth() 中 /api/health/live -> /health/live 的路由不匹配问题
  • 新闻回退 — 内存缓存为空时回退到 last_good_news.json 文件，确保重启后新闻仍可见

用户视觉审查项

以下项目需要用户在实际页面中检查：

  • 标题 — 新闻标题是否完整显示，是否有截断
  • 摘要 — 新闻摘要是否可读，是否溢出
  • 图片相关性 — 图片内容是否与新闻语境基本匹配
  • 裁切 — 图片缩略图是否保持比例，不拉伸变形
  • 字体 — 中文字体是否正常渲染，无方框字
  • 颜色 — 卡片、按钮、徽章颜色是否清晰可辨
  • 留白 — 页面是否有大面积无意义空白
  • 布局 — 侧边栏、内容区、卡片是否对齐一致
  • 操作反馈 — 按钮点击后是否有 toast 提示
  • 导航 — 侧边栏切换页面是否流畅
